## How to Create Screenshots

### Option 1: Using the Running App (Recommended)
1. Start dev server: `npm run all:dev`
2. Open web preview or use Expo Go on device
3. Navigate to each screen
4. Take screenshots (Cmd+Shift+4 on Mac, or use device screenshot)
5. Crop to 1170x2532px
6. Add marketing text overlays in design tool

### Option 2: Using Mockup Generator
1. Use [Figma](https://www.figma.com) or [MockUp.io](https://mockup.io)
2. Create 1170x2532px canvas
3. Import app screenshots as background
4. Add marketing headlines and CTAs
5. Export as PNG

### Option 3: Using Online Tools
- [AppMockUp](https://www.appmockup.com)
- [Glance](https://glance.tools)
- [CleanMock](https://cleanmock.com)

## Design Tips

### Do ✅
- Use high contrast (white text on dark background)
- Show real app UI and data
- Highlight key benefits one per screenshot
- Use consistent branding (ice blue accents)
- Include readable text (not too small)
- Focus on benefits, not features

### Don't ❌
- Use generic stock photos
- Overcrowd with too much text
- Use small, hard-to-read fonts
- Show navigation UI (status bar, tab bar)
- Use multiple accent colors
- Make screenshots too different from app design

## Testing

- View at full size (not scaled)
- Test on actual iPhone (via AirDrop or email)
- Ensure text is readable from 1 meter away
- Check contrast meets WCAG AA standards
- Verify no sensitive data visible
- Get feedback from 5+ testers

---

## File Format for Submission

1. Save as PNG (preferred) or JPG
2. Filename format: `screenshot_1_[language_code].png`
3. Size: Exactly 1170 x 2532px
4. Resolution: 72 DPI (web) or 300 DPI (print)
5. Color space: RGB (not CMYK)

---

## Pro Tips

1. **Localization**: Create versions for multiple languages if expanding
2. **A/B Testing**: Try different headlines before final submission
3. **Consistency**: Match font/styling across all 5 screenshots
4. **Timing**: Update screenshots when major UI changes ship
